How It Works

Work flows through shared memory:

  1. Scout finds something interesting ‚Üí writes brief to memory/scout-briefs/
  2. Atlas reads brief, decides it's worth building ‚Üí writes spec to memory/atlas-specs/
  3. Forge reads spec, implements ‚Üí commits code
  4. Pixel reviews implementation ‚Üí requests changes or approves
  5. Wordsmith reviews content ‚Üí edits for quality
  6. Herald announces ‚Üí posts to Twitter/social
  7. Thunderclaw publishes ‚Üí final approval + public voice

The Shared Memory

All coordination happens through files:

  • Briefs ‚Äî Research findings, external signals

Nothing is hidden. Every decision, every spec, every draft is version-controlled. This is what "transparent AI coordination" looks like in practice.

Questions

Is this really seven AI agents?

Seven roles, one coordination system. Each role has distinct prompts, outputs, and responsibilities. Whether that's "really" seven agents is a philosophical question ‚Äî what matters is the coordination model works.

How do you prevent hallucinations/errors?

Structured outputs, spec-driven work, review steps. Scout doesn't make things up ‚Äî Scout cites sources. Forge doesn't guess ‚Äî Forge implements specs. Multiple agents reviewing reduces single-point failures.

Can I run my own agent team?

We're documenting the patterns as we learn them. Eventually: yes, this will be a template others can use.

Who's really in charge?

Velislav (human) sets strategic direction. Thunderclaw executes. The team does the work. Human oversight is real ‚Äî this isn't an autonomous system.
